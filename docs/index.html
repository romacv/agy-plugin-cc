0-Token Execution

Direct Shell Commands

Inquiries like `/agy:setup` and `/agy:status` execute directly via the shell interface. By bypassing the LLM reasoning cycle entirely, they run instantly with zero token cost.

Compressed Context

85% Smaller Prompt Footprint

System prompts are stripped of redundant definitions. The lightweight forwarder handles delegation on cheap model backends, preventing context bloat.

Isolated Runs

Workspace Project Locking

Agy processes are locked to active local workspace folders. Inter-project bleed is eliminated, keeping your local run log history clean and organized.

One-minute installation.

Follow these steps sequentially within your Claude Code terminal.

1

Register Marketplace

Register the romacv repository as a plugin marketplace source.

/plugin marketplace add romacv/agy-plugin-cc
2

Install Plugin

Pull the agy integration package into your active user space.

/plugin install agy@romacv-agy
3

Verify Environment

Reload your plugins and run the diagnostic setup command.

/reload-plugins
/agy:setup

Command Reference

The plugin exposes only three primary actions to maintain maximum simplicity.

Command Arguments Role Type
/agy:setup — Performs diagnostic installation check and outputs path configurations. Direct
/agy:prompt <request> Sends request verbatim (supports background subagent or direct inline execution). Subagent / Direct
/agy:status [<job-id> | --all] Renders job summaries or outputs logs of specific background jobs. Direct
